## At a glance

A two-phase build where Phase 1 was a tight Figma mockup proving the user journey to the client, and Phase 2 was the development handoff. I owned scope across both phases and the bridge between them.

## The problem

The client wanted certainty before committing engineering budget — totally reasonable, but the usual risk is that a Figma mockup looks decisive while hiding a hundred decisions that engineering has to re-litigate. Phase 1's job wasn't to make a pretty file; it was to remove ambiguity so Phase 2 could ship without backtracking.

## My role

- **Scope owner** across both phases — kept the mockup honest about what was decided versus what was deferred.
- **Documentation lead** — requirements, screen flows, and design notes captured in a project knowledge base.
- **Handoff coordinator** — Figma → engineering with annotated specs.

## Approach

- Treated the mockup as a contract, not a canvas: every screen carries a comment thread of decisions and a "what we're deliberately not deciding yet" list.
- Built a screen index that maps every Figma frame to a documented spec — so during Phase 2 development, engineering doesn't have to guess which frame corresponds to which behaviour.
- Used Figma credentials in a shared knowledge file so the dev team could read source-of-truth without re-exporting assets.

## What shipped

- **Phase 1**: Complete Figma mockup + specs + screen flow documentation
- **Phase 2**: Development handoff package; engineering picked up with no follow-up ambiguity meetings

## Outcome

- Phase 2 began on schedule with zero scope-rework sprints — uncommon when a client validates on mockups.
- The knowledge structure (PRD, Figma index, and specs in one folder) became a template that I later reused for Artlive and Bside.

## What I'd do differently

Lock the **content model** in Phase 1, not Phase 2. We left some entity-relationship questions for engineering to surface — they did surface them, but later than ideal. Modelling data alongside screens during mockup phase pays back ~1.5x in dev velocity.
